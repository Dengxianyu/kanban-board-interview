# 面试任务：实现卡片依赖关系功能

## 需求背景

产品经理希望在看板中加入卡片之间的依赖关系，让团队能更好地管理任务之间的前后置关系。

## 功能需求

1. **设置依赖**：创建或编辑卡片时，可以选择该卡片"依赖的卡片"（支持多选）
2. **拖拽约束**：如果一张卡片有未完成的依赖（依赖的卡片不在 Done 列），则不能将其拖拽到 Done 列
3. **状态显示**：在卡片上显示依赖状态，如"2/3 依赖已完成"
4. **阻塞提示**：用户需要能一眼看出哪些卡片是被阻塞的（有未完成的依赖）

## 技术要求

- 在现有项目架构基础上实现（保持 hooks + context 的模式）
- 依赖数据需要持久化到 localStorage
- 确保不影响现有功能的正常使用

## 时间

30 分钟
