# 看板面试题

> 面试官内部文档。两道题独立使用，每次面试选一道。

## 环境准备

```bash
pnpm install
pnpm dev       # http://localhost:5173
```

技术栈：React 19 + TypeScript + Vite，零第三方运行时依赖。

候选人到场后给 5 分钟熟悉代码结构，然后口述题目要求（不给书面文档）。建议总时长 60–90 分钟。

## 题目一：卡片截止日期（100 分）

### 口述给候选人

> 请为看板卡片添加"截止日期"功能：创建/编辑卡片时可以设置截止日期，卡片上展示日期，过期的卡片需要有明显的视觉区分，筛选栏支持按到期状态筛选，同列卡片支持按截止日期排序。

### 评分标准

| 模块 | 分值 | 满分要求 | 基本得分线 |
|------|------|---------|-----------|
| 类型定义 | 10 | `Card` 类型正确扩展 `dueDate?: number`，`FilterState` 相应扩展 | 类型加了但不够准确（如用 string）(5) |
| 表单交互 | 20 | 新建 / 编辑均支持日期选择，交互自然，可清除 | 只在新建支持 (10)；选择器能用但无法清除 (15) |
| 卡片展示 | 20 | 日期格式化可读，过期卡片有强视觉标识（边框 / 背景 / 图标），即将到期(≤3天)有弱提醒 | 只展示日期无过期样式 (8)；有过期样式但无即将到期 (15) |
| 筛选功能 | 25 | `FilterBar` 新增到期状态筛选（全部 / 已过期 / 即将到期 / 未设置），与现有标签、优先级筛选正确组合 | 能筛选过期但不能组合 (12)；能组合但漏了"未设置"筛选 (20) |
| 排序功能 | 15 | 列内卡片可切换按截止日期升序排列，无截止日期的排在最后 | 能排序但无截止日期的处理不对 (8) |
| 代码质量 | 10 | 类型安全无 `any`，逻辑复用合理，命名清晰，无冗余渲染 | 有少量 `any` 或硬编码 (5) |

### 观察要点

- **类型优先**：是否先改 `types/index.ts` 再写逻辑？还是边写边补？
- **日期处理**：用原生 `<input type="date">` 还是自己造？（原生即可，不扣分）
- **过期判定**：是否考虑了时区问题？用 `Date.now()` 还是取当天零点？
- **筛选组合**：多个筛选条件是 AND 还是 OR？能否清晰解释选择？
- **排序切换**：是全局开关还是每列独立？状态放在哪？

## 题目二：撤销 / 重做（100 分）

### 口述给候选人

> 请为看板添加"撤销/重做"功能：所有卡片操作（新增、编辑、删除、拖拽移动）都可以撤销和重做。需要支持 Cmd/Ctrl+Z 撤销、Cmd/Ctrl+Shift+Z 重做。工具栏上需要展示撤销/重做按钮，并反映当前是否可操作。

### 评分标准

| 模块 | 分值 | 满分要求 | 基本得分线 |
|------|------|---------|-----------|
| 数据结构设计 | 25 | 方案合理（快照栈 / 命令模式均可），能口头解释权衡 | 有方案但说不清为什么这样选 (12)；方案有明显缺陷 (8) |
| 核心逻辑 | 30 | 四种操作（增删改移）撤销 / 重做均正确，新操作清空 redo 栈 | 覆盖 2-3 种操作 (15-20)；只覆盖 1 种 (8) |
| 快捷键 | 15 | 正确绑定 Cmd/Ctrl+Z 和 Cmd/Ctrl+Shift+Z，输入框聚焦时不触发 | 快捷键能用但未处理输入框冲突 (10)；只实现了撤销没有重做 (7) |
| UI 集成 | 15 | 按钮置于 Toolbar，disabled 状态正确，点击后看板实时更新 | 按钮能用但 disabled 状态不对 (10)；没加按钮只有快捷键 (5) |
| 边界处理 | 10 | 历史上限（如 50 步）、空栈保护、连续快速操作不出错 | 有空栈保护但无历史上限 (5) |
| 代码质量 | 5 | Hook 封装独立、类型完备、与现有代码集成干净 | 逻辑散落在组件中 (2) |

### 观察要点

- **方案选型**：快照 vs 命令模式？快照简单但内存开销大，命令模式更优雅但实现复杂。重点是**能否说清权衡**。
- **与现有架构的集成**：是否新建了独立的 hook/context？还是直接侵入 `useBoard`？
- **LocalStorage 同步**：撤销后 localStorage 是否也回滚了？
- **重做栈清除**：新操作后是否正确清空了 redo 栈？
- **输入框冲突**：在 CardForm 的 input 里按 Ctrl+Z 应该是撤销输入，而不是撤销看板操作。

## 补充说明

- 候选人可以使用 AI 工具，但需能解释每行代码的意图。如果发现候选人无法解释自己写的代码，该模块 **直接 0 分**。
- 两道题难度基本持平：题目一偏功能广度，题目二偏架构深度。根据岗位侧重选择。
- 如果候选人提前完成，可追加提问：「如果卡片数量达到 1000+，你会怎么优化？」
