# 评分流程

1. 运行 e2e 测试：`pnpm test:e2e`
2. 分析测试结果
3. 检查代码改动：`git diff main..HEAD --stat`
4. 检查代码质量

## 评分维度

### 基本功能测试（18 分）
Tests 1-5，每个 3.6 分
- 依赖选择 UI
- 创建时设置依赖
- 依赖状态显示
- 拖拽约束（未完成依赖不能到 Done）
- 依赖完成后可拖到 Done

### 边界情况（21 分）
Tests 6-10，每个 4.2 分
- 循环依赖检测
- 级联影响（依赖从 Done 移走后的告警）
- 删除被依赖卡片的清理
- 筛选状态下的依赖约束
- 旧数据兼容性

### 工程质量（9 分）
Tests 11-12
- 依赖数据持久化 = 4 分
- 无功能回归 = 5 分

### 代码规范（12 分）
- 遵循项目 hooks + context 模式 = 5 分
- TypeScript 类型正确 = 3 分
- 改动范围合理 = 2 分
- 无新增 console.log/warn = 2 分

## 评级标准（60 分制）

- A（54-60）：优秀 - 功能完整，边界处理好，代码质量高
- B（42-53）：良好 - 核心功能完成，部分边界处理
- C（30-41）：及格 - 基本功能可用，边界情况不足
- D（0-29）：不及格 - 功能不完整或严重缺陷

## 注意事项

Feature 测试的 UI 选择器使用了启发式匹配（如 `[data-testid*="depend"]`）。如果候选人的实现使用了不同的命名约定，测试可能需要人工调整选择器后重新运行。评分时请先检查候选人的 UI 实现，必要时更新测试选择器。
